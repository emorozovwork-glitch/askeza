# askeza

Приложение для отказа от вредных привычек. Помогает отслеживать время без привычки, видеть снижение риска рецидива и зарабатывать жетоны-достижения за пройденные этапы.

## Стек и платформа

- **Сейчас:** Single-file PWA (HTML + CSS + JS в одном `index.html`)
- **Целевая платформа:** Нативное iOS-приложение (App Store)
- **Текущий статус:** Прототип для личного использования через браузер iPhone (Add to Home Screen)
- **Данные:** localStorage, синхронизация не нужна на текущем этапе

## Что уже реализовано (v0.1)

### Ядро
- Добавление нескольких привычек с именем, эмодзи, цветом и датой старта
- Переключение между привычками через боковое меню
- Хранение данных в localStorage
- Редактирование и удаление привычки (с сохранением лучшего жетона)

### Главный экран
- **Таймер-бары** — анимированные полосы для каждой единицы времени (секунды → годы), обновляются в реальном времени
- **Счётчик риска рецидива** — снижается по научной формуле с динамическим количеством знаков после запятой
  - Курение: `99/√(1 + дни) + 1` (быстрое снижение)
  - Алкоголь: `88/(1 + дни)^0.30 + 4` (источники: NIAAA 2023, PMC)
  - Фазы риска: Критическая → Опасный период → Становится легче → Безопасная зона
- **Вдохновляющая цитата** с кнопкой обновления (19 цитат на русском)

### Система жетонов (Tokens)
- 20 этапов-вех: от "Начало" (60 сек) до "1 год"
- Ранги: XX → I (как боевые звания)
- Жетоны зарабатываются автоматически при достижении вехи
- Отображение активных и архивных жетонов
- Счётчик всех жетонов в шапке (🏆)
- При сбросе привычки — лучший жетон сохраняется в архив

### Пресеты привычек
- Курение, Алкоголь, Соцсети, Сахар, Азартные игры
- Каждый пресет имеет свой цвет

### UX и дизайн
- Тёмная тема в стиле iOS (цветовые переменные из iOS system palette)
- Safe area insets для iPhone с вырезом
- Анимации появления элементов (fadeUp, bounce, confetti)
- PWA-метатеги: `apple-mobile-web-app-capable`, standalone display

## Версионирование

**Важно:** при **каждом** изменении (даже мелком) обязательно:

1. Поменять версию в `index.html` — в топбаре: строка с `askeza <span ...>vX.Y</span>`
2. Поменять версию в `CLAUDE.md` — строка `**Текущая версия**` ниже
3. Запушить на GitHub (`git add -A && git commit -m "vX.Y ..." && git push`)

Цель: пользователь должен с iPhone сразу видеть, обновилась ли версия.

**Текущая версия: `v1.1.2`**

Схема: `v[major].[minor]` — minor меняется при мелких правках (0.81, 0.82…), десятая доля — при значительных обновлениях (0.9, 1.0…).
